# Welcome

I am Xihao Yang, a master's student in Software Engineering at Keio University. My current research interests are in Automated Program Repair, software testing, and AI-based software engineering.

My research focuses on how to evaluate whether automatically generated patches are truly correct. In Automated Program Repair, a patch may pass the existing test cases but still fail to preserve the intended behavior of the program. This is known as the overfitting patch problem. I am interested in learning-based and execution-aware methods for patch correctness assessment.

# Research Interests

* Automated Program Repair
* Patch correctness assessment
* Software testing and debugging
* AI for Software Engineering
* Program behavior analysis
* Code representation learning

# Current Work

I am currently exploring execution-aware signals for learning-based patch correctness assessment. The basic idea is to go beyond only comparing source code changes and also consider how patched programs behave during execution.

This direction is motivated by a practical problem: existing test suites are often incomplete, so passing all tests does not always mean that a generated patch is semantically correct. By combining code-level information with execution behavior, I aim to study whether patch correctness prediction can become more reliable and interpretable.

# Projects

My recent work includes reproducing and studying methods related to learning-based patch correctness assessment, including approaches based on program representations, code models, and test execution behavior. I am also interested in building small experimental frameworks to compare correct and overfitting patches under different analysis settings.
